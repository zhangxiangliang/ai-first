## 摘要

本项目要准备一场 10-20 分钟的中文分享，面向即将毕业的大学生。主题是：

> AI First 到 AI 100%：当 AI 写完所有代码，软件工程师还剩下什么？

分享风格要幽默、真诚、实用。核心结构是同一个问题回答两次：

1. 编码 100% 交给 AI，人做什么？  
   第一次回答：人要学会 AI-first 工作流。
2. 编码 100% 交给 AI，人做什么？  
   第二次回答：人要做自己，主动尝试、验证想法、积累判断和作品。

## 已确认事实

- 听众主要是即将毕业的大学生。
- 分享时长为 10-20 分钟。
- 交付物包括 PPT 大纲、完整 HTML PPT、逐字稿。
- Claude 和 Claude Code 可以直接讲。
- AI 工作流教学示例使用这个 PPT 项目本身。
- 这个设计可以形成闭环：PPT 由流程产生，PPT 又讲这个流程。
- AI 写文章/PPT 的小生意要表达两点：
  - 当时 AI 的能力已经不错。
  - 不要太早替市场下结论，小众需求也可能有真实市场。
- 小生意段落可以鼓励学生尝试和验证想法。
- 公司例子可以讲：团队从 20 人左右到 4 人，也出现了多线程、多窗口使用 AI 完成任务的情况。
- 原始输入来自录音转文字，存在转写错误。
- 本项目文档优先使用中文，方便后续作为演示素材。

## 分享对象上下文

听众是即将毕业的大学生，他们可能关心：

- 我该做前端、后端、测试、运维，还是别的方向？
- 我该学 Java、Python，还是 AI 工具？
- AI 会不会让程序员失业？
- 我现在还能做什么积累？
- 怎么用 AI 做出能展示的项目？
- 有没有小机会可以尝试？

## 内容上下文

用户个人经历大致可以整理为：

1. **搜索框阶段**  
   早期 AI 像一个更聪明的搜索框，能更快给答案，但没有真正改变工作流。

2. **小助手阶段**  
   AI 可以写简单代码、测试用例、翻译、文章、PPT，开始进入交付过程。

3. **小生意阶段**  
   用户通过 AI 辅助写文章和 PPT，每月约一两千元。这个例子说明 AI 已经可用，也说明市场需求并不总是由“顶尖质量”决定。

4. **Claude Code / vibe coding 阶段**  
   AI 更像“手和脚”，用户开始在做任何事情前先思考：哪一部分可以让 AI 参与？

5. **AI-first 工作流阶段**  
   人的价值从“亲手写代码”迁移到：
   - 讲清楚问题。
   - 提供上下文。
   - 定义验收标准。
   - 判断结果对不对。
   - 把经验沉淀为项目记忆。

6. **职业和人生建议阶段**  
   当执行成本下降，学生更应该主动尝试、验证想法、积累作品，而不是只等待标准答案。

## 工作流演示上下文

本项目本身就是工作流示例：

| 阶段 | 在本项目中的体现 |
|------|------------------|
| raw-input | 用户提供录音转文字稿。 |
| discovery | 提取主题、受众、叙事主线、风险和问题。 |
| requirements | 明确时长、听众、交付物、风格、验收标准。 |
| context | 记录纠错、确认事实、叙事骨架。 |
| tech-spec | 设计 HTML PPT 的实施方案。 |
| implementation | 用户审核通过后，写大纲、逐字稿、HTML PPT。 |
| review | 检查时间、可读性、叙事和演示效果。 |
| wiki | 将稳定方法论沉淀为可复用知识。 |

## 叙事骨架

| 部分 | 作用 | 内容要点 |
|------|------|----------|
| 开场 | 建立记忆点 | 张大柱、海报头像、幽默开场。 |
| AI 使用旅程 | 建立可信度 | 搜索框 -> 小助手 -> Claude Code / vibe coding。 |
| 小生意 | 说明 AI 可用和市场复杂性 | 写文章/PPT，1-2 千/月，小众不一定小。 |
| 第一次提问 | 进入工作流主线 | 编码 100% 交给 AI，人做什么？ |
| 第一次回答 | AI 工作流程教学 | 需求、上下文、设计、验收、项目记忆。 |
| 原汤化原食 | 用 PPT 项目做例子 | 展示当前工作流如何产出 PPT。 |
| 第二次提问 | 从工作方法转向职业人生 | 同一个问题再问一次。 |
| 第二次回答 | 给学生建议 | 做自己，主动尝试，验证想法，积累作品。 |
| 结尾 | 强化记忆点 | 可能忘记概念，但记住“张大柱”。 |

## 技术上下文

- 最终 PPT 方向是 HTML。
- 推荐使用浏览器可演示的幻灯片形式。
- 可以考虑 Reveal.js，也可以考虑自定义 HTML/CSS/JS。
- 如果现场网络不稳定，应考虑离线可用。
- 最终文件建议放在 `workspaces/ppt/implementation/`。
- 真正生成 PPT 前需要用户明确要求。

## 假设

- 10-20 分钟适合约 10-14 页 PPT。
- 口语化中文更适合这次分享。
- 听众会更容易记住故事、例子和演示，而不是记住概念名词。
- “AI 100%”应作为有冲击力的假设表达，不应说成绝对事实。
