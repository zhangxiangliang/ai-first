我每天的流程

Raw Input→ Discovery→ Requirement→ Tech Spec→ Implementation→
Review→ Wiki

这套流程可以和 AI 一起协同开发。

例子 · 现身说法

你正在看的这份 PPT，
就是这么来的

这不是案例，这是现场证据。

Raw Input

先把最粗糙、最口语、还带错别字的原始想法保存下来。

Discovery

AI 先帮我找问题、拆主线、追问缺失的信息。

Requirement

把时间、听众、交付物、风格和验收标准确认下来。

Tech Spec

先定格式、文件位置、依赖和边界，再进入真正制作。

Implementation

先写大纲和逐字稿，再生成真正能演示的 HTML PPT。

Review

检查页数、文案、图片、动效、离线依赖和文档状态。

Wiki

最后把稳定的方法沉淀下来，让下一个项目不用从零开始。

项目，也需要"记忆"

把需求、BUG、每个决定沉淀成 Wiki。
任何时间、任何人、任何 AI，都能拿到答案。
